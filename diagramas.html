Sistema de Gestión de Pedidos para Restaurantes

03 — Modelado

Diagramas del Sistema

Modelo Entidad-Relación (conceptual, lógico y físico), diagrama de clases, diccionario de datos y componentes de seguridad en la base de datos.

📐 Modelo Entidad-Relación (MER) — Conceptual

El modelo conceptual representa las entidades principales del sistema, sus atributos más relevantes y las relaciones entre ellas a alto nivel, sin detallar tipos de datos ni restricciones físicas. Sirve como base para la comunicación entre analistas y partes interesadas.

🔧 Modelo Lógico — Tablas y Relaciones

El modelo lógico transforma el modelo conceptual en tablas con columnas, tipos de datos, claves primarias (PK) y claves foráneas (FK). Define las cardinalidades y reglas de integridad referencial entre las entidades.

🗄️ Modelo Físico — Implementación en MySQL

El modelo físico muestra la implementación real de la base de datos en phpMyAdmin con tipos de datos específicos de MySQL (INT, VARCHAR, DECIMAL, TIMESTAMP, TEXT, BOOLEAN), restricciones NOT NULL en todas las PK y FK, y la estructura completa de cada tabla.

Procedimientos Almacenados

sp_RegistrarPedido: El motor MySQL valida la disponibilidad del producto antes de procesar la orden, evitando INSERTs directos desde el backend.

Triggers
